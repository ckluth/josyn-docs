JOSYN Toolbox

This repo contains maintainer tooling for the JOSYN platform. It is not a platform component. The platform never references it. It carries none of the platform's obligations.

Purpose

josyn-toolbox is the maintainer's collection of operational scripts and developer utilities:

  • Deployment — full local deployment pipeline for the JOSYN stack
  • Machine synchronisation — keeping all repositories in sync across machines
  • Developer tools — code generators and documentation utilities used during development

Tools here are stable and regularly executed. They are maintained to a higher standard than playground experiments, but they are not platform infrastructure.

What this repo is not

  • Not a platform component
  • Not maintained to platform standards
  • Not a source of architectural truth
  • Not referenced by any platform repo — in any form

The one hard rule

The platform must never take any dependency on this repo.

Repository layout

josyn-toolbox/
├── cfg/                        ← shared configuration: repo list, root detection
├── git-tools/                  ← machine-sync scripts: clone, push-all, pull-all, repo-status
├── arg-gen/                    ← .NET tool: generates local-arguments scaffold for deployed jobs
├── site-builder/               ← documentation pipeline: index → assemble → HTML site
│   ├── site-builder-indexer/   ← scans repos, produces docs-index.json
│   ├── site-builder-assembler/ ← stages files by logical structure, generates landing/search
│   └── site-builder-renderer/  ← .NET tool: markdown folder → static HTML site
├── deploy/                     ← deployment pipeline: full local deploy of the JOSYN stack
└── convenience-scripts/        ← one-shot utilities: NuGet total rebuild, etc.

Tools

cfg/

Shared configuration used by all tools in this repo:

  • cfg-detect-root.cmd — maps hostname to the local DevGit root path
  • cfg-repos-list.cmd — ordered list of all JOSYN repositories (for git sync)
  • cfg-docs-repos-list.cmd — repositories included in the documentation index scan

git-tools/

Scripts for bootstrapping a fresh machine and keeping all JOSYN repositories in sync when switching between machines (Planet Home-Office ↔ Planet Company).

See git-tools/README.md for full documentation.

deploy/

PowerShell deployment pipeline that builds and publishes the full JOSYN stack to the local developer machine. Covers NuGet pack, JAPServer, CLI, adapters, and demo job.

See deploy/README.md for full documentation.

arg-gen/

A .NET 10 console tool that generates the full local-arguments\ scaffold for a deployed job — arguments-default.ini, _launcher.cmd, and arguments-default.cmd. Called by deploy/deploy-maintainer.ps1 after each job publish step.

local-arguments\ is a deploy-time artefact. It must not be committed to job repos.

Usage:

dotnet run --project arg-gen -- <job-exe-path> --cli-path <cli-exe-path> [--output-dir <dir>]

site-builder/

Three-part toolchain that assembles the scattered markdown documentation across all JOSYN repositories into a single, navigable HTML site in josyn-docs.

Part Entry point What it does
site-builder-indexer site-builder-indexer.cmd Scans repos → docs-index.json
site-builder-assembler site-builder-assembler.cmd Stages files + calls renderer
site-builder-renderer Called by assembler; also standalone Markdown folder → HTML site

See site-builder/README.md for the full pipeline overview.
