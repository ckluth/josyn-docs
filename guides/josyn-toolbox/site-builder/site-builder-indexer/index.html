site-builder-indexer

Scans all JOSYN repositories for markdown files and produces docs-index.json — the master documentation index consumed by the rest of the site-builder pipeline.

Part of the site-builder toolchain.

What it does

  1. Walks each configured JOSYN repo, finding every *.md file (excluding .git/ and docs/generated/).
  2. For each file, extracts mechanical fields from the file system and git history:
    • name — first H1 heading in the file
    • repo — repository name
    • path — path relative to the repo root (forward slashes)
    • folder-context — top-level folder, or "" for root-level files
    • created — date of the oldest git commit adding the file
    • touched — date of the most recent git commit touching the file
  3. Preserves any existing semantic fields (type, perspective, state, summary, enriched-at) from a previous run. These are filled by the AI enrichment pass and must not be overwritten.
  4. Removes entries for files that no longer exist on disk.
  5. Writes the updated index to josyn-platform/docs/docs-index.json.
  6. Writes two generated artefacts to josyn-platform/docs/generated/:
    • docs-index.md — markdown table of all indexed documents, grouped by repo
    • docs-index-report.md — execution report (counts, new/retained/removed)

Usage

site-builder-indexer.cmd

No arguments. All paths are resolved by cfg/cfg-detect-root.cmd and cfg/cfg-docs-repos-list.cmd.

Configuration

Which repos are scanned is controlled by josyn-toolbox/cfg/cfg-docs-repos-list.cmd. This is a separate list from cfg-repos-list.cmd (which covers git-tools operations). Edit it to add or remove repos from the index.

Output

josyn-platform/docs/docs-index.json — structure:

{
  "generated": "2026-06-07T...",
  "documents": [
    {
      "name": "Architecture Overview",
      "repo": "josyn-platform",
      "path": "architecture/overview.md",
      "folder-context": "architecture",
      "created": "2026-05-29T...",
      "touched": "2026-06-05T...",
      "type": "architecture-overview",
      "perspective": ["maintainer-architect", "maintainer-developer", "agent"],
      "state": "settled",
      "summary": "Explains the platform structure, runtime flow, and dependency graph.",
      "enriched-at": "2026-06-07T..."
    },
    ...
  ]
}

Semantic fields (type, perspective, state, summary, enriched-at) are null for newly discovered documents. They are filled by the AI enrichment pass — see josyn-platform/docs/enrich-docs-index-prompt.md.

Semantic field vocabulary

Defined in josyn-platform/architecture/docs-index-vocabulary.md.

Field Values
type architecture-overview, architecture-detail, component-architecture, solution-architecture, decision, reference, manual, working-note, overview, repo-summary, plan, agent-instruction
perspective maintainer-architect, maintainer-developer, maintainer-operator, job-developer, agent, stakeholder
state thinking, in-progress, settled, superseded

Dependencies

  • pwsh (PowerShell 7+)
  • git (must be in PATH)
  • All scanned repos cloned and accessible on disk
