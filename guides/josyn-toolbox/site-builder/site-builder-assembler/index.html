site-builder-assembler

Reads docs-index.json and site-map.yaml, assembles a staging folder with a logical site structure, generates a landing page and search page, then optionally calls site-builder-renderer to produce the final HTML site.

Part of the site-builder toolchain.

What it does

  1. Parses site-map.yaml — tree, mapping rules, overrides, reading paths.
  2. Loads docs-index.json — the full list of indexed documents with semantic metadata.
  3. For each document, determines its target path in the staging folder:
    • Overrides with skip: true — excluded entirely (matched by repo + path).
    • Overrides with to: — placed at the exact specified tree path.
    • Mapping — first matching rule wins; places the document at the mapped tree path.
    • Catch-all — documents matching nothing land in the designated catch-all node.
  4. Clears the staging folder and copies all matched source files to their target paths.
    Staged path pattern: {tree-path}/{repo}/{original-path}.
  5. Generates a README.md per tree node — lists sub-nodes and documents in that node.
  6. Generates README.md at the staging root — the landing page with perspective-based reading path cards.
  7. Generates search.md — a full document list with a JS text filter.
  8. Calls site-builder-renderer (dotnet run) unless -NoBuild is set.

Usage

site-builder-assembler.cmd

Stages only — does not call the renderer. Inspect the staging folder at temp\josyn-site-staging\, then run site-builder-renderer.cmd to produce the HTML.

To stage and render in one step:

pwsh -File .internal\site-builder-assembler.ps1 -SiteMap <path>

Configuration: site-map.yaml

Lives in josyn-platform/docs/site-map.yaml. Editorial configuration — the assembler reads it but never modifies it.

For a full authoring guide see site-map-authoring.md.

site block

site:
  title: "JOSYN Documentation"
  docs-index: "C:/DevGit/josyn-platform/docs/docs-index.json"
  repos-root: "C:/DevGit"
  output: "C:/DevGit/josyn-docs"
  staging: "C:/DevGit/temp/josyn-site-staging"

tree block

The human-authored folder hierarchy. Node names become folder names in staging and labels in the site navigation. Depth is free. One node may be marked catch-all: true.

tree:
  - architecture:
      - platform
      - solution-architecture
      - components
  - decisions:
      - platform
      - backend
  - repos
  - guides
  - notes
  - unsorted:
      catch-all: true

mapping block

Flat list, evaluated top-to-bottom, first match wins. Each entry has criteria fields (type, repo, folder-context, state) and either to: (tree path) or skip: true. Criteria fields are AND. Wildcards (*) supported in string values.

mapping:
  - { type: agent-instruction, skip: true }
  - { type: architecture-overview, to: architecture/platform }
  - { type: decision, repo: josyn-backend, to: decisions/backend }

overrides block

Per-file exceptions. Matched by repo + path. Applied before mapping.

overrides:
  - { repo: josyn-platform, path: README.md, to: repos }
  - { repo: josyn-platform, path: MAINTAINERS.md, skip: true }

reading-paths block

Hand-curated starting guides per perspective. Shown on the landing page as cards.

reading-paths:
  - perspective: stakeholder
    label: "Stakeholder"
    intro: "Understand what JOSYN is and where it is heading."
    links:
      - { repo: josyn-platform, path: README.md }
      - { section: decisions }

Staging folder layout

{staging}/
  README.md                              ← generated landing page
  search.md                              ← generated search page
  architecture/
    README.md                            ← generated node index
    platform/
      README.md                          ← generated node index
      josyn-platform/
        architecture/
          overview.md                    ← copied source file
  decisions/
    platform/
      README.md
      josyn-platform/
        decisions/
          ADR-001-platform-naming.md
  repos/
    README.md
    josyn-foundation/
      README.md
    josyn-backend/
      README.md

CLI parameters

Parameter Default Description
-SiteMap (required) Path to site-map.yaml
-ReposRoot from site-map Override repos-root
-StagingDir from site-map Override staging directory
-NoBuild false Stage only; do not call the renderer

Dependencies

  • pwsh (PowerShell 7+)
  • dotnet SDK — for calling site-builder-renderer
  • josyn-platform/docs/docs-index.json — up-to-date index (run indexer first)
  • josyn-platform/docs/site-map.yaml — assembly configuration
  • All source repos accessible on disk under repos-root
