site-builder-renderer

A C# tool that converts a folder of markdown files into a self-contained, static HTML site with a sidebar navigation, responsive design, and a shared CSS stylesheet.

Part of the site-builder toolchain. Normally called by site-builder-assembler — run it standalone only for single-folder use cases.

What it does

  1. Discovers all *.md files in the source directory (recursive).
  2. Builds a navigation tree: one section per top-level subfolder; pages within each section.
  3. For each page:
    • Parses markdown using Markdig (with advanced extensions: tables, code, footnotes, etc.)
    • Rewrites .md links to .html links (preserving relative paths)
    • Wraps content in the HTML template with the generated nav sidebar
  4. Writes one .html file per .md file; README.md becomes index.html.
  5. Writes a single site.css to the output root.

Usage

Via assembler (normal)

The assembler calls the renderer automatically. No manual invocation needed.

Standalone

For rendering a single folder (e.g. solution-architecture only):

site-builder-renderer.cmd

Edit the SOURCE, OUTPUT, and TITLE variables at the top of the .cmd file.

Or call directly:

dotnet run --project DocGenerator.csproj -- <source-dir> <output-dir> [<site-title>]

Arguments

Position Name Description
1 source-dir Directory containing .md files to render
2 output-dir Directory where HTML files are written (created if absent)
3 site-title (optional) Title shown in the nav header and <title> tags

Navigation structure

The nav sidebar is derived from the folder structure of the source directory:

  • A root README.md (-> index.html) becomes the Home link.
  • Each top-level subfolder becomes a nav section.
  • A README.md inside a subfolder becomes the section's Overview link.
  • All other pages in the subfolder are listed as children under the section.

Pages nested deeper than one level still appear in the nav under their top-level section.

Link rewriting

All relative .md links in the source markdown are rewritten to .html links in the output. Directory-style links (ending with /) are resolved to the index.html in that directory. External and anchor-only links are left unchanged.

Source files

File Responsibility
Program.cs Argument validation, entry point
SiteBuilder.cs File discovery, nav tree construction, orchestration
PageRenderer.cs Markdig parsing, link rewriting, HTML assembly
HtmlTemplate.cs HTML template and CSS (all visual concerns)

Dependencies

  • .NET 8 SDK
  • NuGet: Markdig (markdown parsing with advanced extensions)
